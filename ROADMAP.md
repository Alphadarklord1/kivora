# Kivora Roadmap

This roadmap is for stable-beta execution, not speculative feature sprawl.

## Product Correction

Kivora should not try to win by becoming a clone of Quizlet, Duolingo, or a generic AI chatbot.

We should win by being:

- a better study workspace
- better at turning raw material into usable outputs
- stronger on desktop and offline use
- more honest about what is collaborative, AI-assisted, or still lightweight

That changes the roadmap in a few important ways:

- incorporate now:
  - cold-start onboarding
  - starter/shared content library
  - pruning or demoting weak tools
- make different:
  - groups should be positioned as shared study spaces first, not fake real-time competition
  - AI should win on grounded workflow and integrated study flow, not on pretending to beat Claude at raw writing
- delay, but do not ignore:
  - teacher/classroom tooling
  - social graph, discovery, and competitive retention systems

## Now

- stabilize desktop + web beta behavior
- tighten planner, analytics, and workspace flows
- finish auth reliability and Google sign-in setup
- keep offline Mini usable by default
- keep Arabic/RTL quality acceptable on active surfaces
- add a real cold-start path so a student is not blocked if they do not already have a PDF
- seed the app with starter study packs, templates, and reusable shared content
- improve import paths from notes, PDFs, pasted text, and YouTube so users do not start from zero
- prune, hide, or re-scope weak tools that overpromise
- make Groups read honestly as async shared study spaces unless a real live mode exists

## Next

- planner drag/drop and stronger calendar interaction
- stronger workspace/file handling and preview flow
- better math input and equation UX
- improved AI result traceability and source messaging
- release automation hardening for desktop + model assets
- improve AI quality where it matters most:
  - source-grounded research
  - writing feedback
  - assignment structuring
- add curated or community-shared study packs with quality control instead of an unfiltered content dump
- strengthen search and discovery inside Library around useful study material, not social noise

## Later

- deeper collaboration features with actual live value
  - live group review
  - host-led quiz sessions
  - shared study rooms
- richer local AI runtime controls
- stronger analytics recommendations
- higher-fidelity mobile browser support
- teacher/classroom tools
  - assignments
  - class rosters
  - student progress reporting
- social and discovery systems
  - public profiles
  - trending/shared decks
  - leaderboards
  - following

## Not planned during beta

- broad feature expansion outside the supported surfaces
- re-enabling unfinished encryption password flows
- unstable or placeholder tools exposed in the main UI
- pretending weak tools are flagship features before they are genuinely strong
- trying to outrun established incumbents by copying their biggest-network features too early
