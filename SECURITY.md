# Security Policy

## Supported scope

Kivora is currently in stable beta. Security issues affecting these areas should be reported privately:

- authentication and session handling
- guest-mode access boundaries
- OAuth configuration
- file access and share links
- offline model download integrity
- desktop runtime / local AI execution

## Please do not open public issues for security vulnerabilities

Use a private channel first:

- GitHub Security Advisories for this repository, if enabled
- or contact the maintainer directly before public disclosure

## What to include

- affected route, file, or feature
- exact reproduction steps
- whether the issue affects web, desktop, or both
- screenshots or logs if relevant
- impact assessment

## Disclosure expectations

- Give the maintainer time to reproduce and patch the issue.
- Do not publish exploit details before a fix or mitigation exists.
