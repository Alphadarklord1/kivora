# Support

Use the repository in this order:

## 1. Questions and collaboration

Use GitHub Discussions if enabled. If Discussions are not enabled, open an issue only if there is a concrete bug, feature request, or task.

## 2. Bugs

Open a bug report and include:

- runtime: web or desktop
- environment: macOS, Windows, browser
- exact page or API route
- screenshot or console/network error
- whether guest mode or sign-in was used

## 3. Features / team work

Open a feature request or draft issue with:

- the user problem
- the target surface
- expected behavior
- constraints or rollout concerns

## 4. Security

Do not file public issues for vulnerabilities. Follow `SECURITY.md`.
