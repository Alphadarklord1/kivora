# Code of Conduct

## Expected behavior

- Be direct and respectful.
- Critique code and decisions, not people.
- Keep bug reports and reviews concrete.
- Prefer evidence, repro steps, and code references over vague opinions.

## Unacceptable behavior

- Personal attacks
- Harassment
- Publicly posting security vulnerabilities before a fix exists
- Dismissing contributors without technical reasoning

## Maintainer standard

Maintainers should keep collaboration predictable:

- explain merge/reject decisions clearly
- keep scope boundaries explicit
- avoid leaving contributors blocked without feedback

## Reporting

For community conduct problems, contact the maintainer privately.
